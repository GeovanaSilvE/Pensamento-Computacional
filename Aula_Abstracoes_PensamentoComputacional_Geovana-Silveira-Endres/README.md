# 🏥 MedSync — Sistema Inteligente de Gestão Hospitalar

## 📚 Sobre o Projeto

O **MedSync** é uma aplicação web conceitual voltada para gestão hospitalar inteligente, criada com o objetivo de centralizar e automatizar processos médicos e administrativos em um único sistema digital.

O projeto foi desenvolvido para a disciplina de **Pensamento Computacional**, utilizando conceitos de:
- abstração computacional;
- decomposição;
- reconhecimento de padrões;
- algoritmos;
- modelagem lógica.

A proposta do sistema é solucionar problemas comuns encontrados em hospitais e clínicas, como:
- desorganização de prontuários;
- dificuldade no gerenciamento de consultas;
- excesso de processos manuais;
- falhas administrativas;
- descentralização de informações.

Os conceitos aplicados seguem os princípios apresentados na aula sobre abstração computacional. :contentReference[oaicite:0]{index=0}

## 💡 Sistema Proposto

O MedSync é um sistema hospitalar inteligente responsável por:
- cadastrar pacientes;
- organizar consultas;
- armazenar prontuários digitais;
- registrar exames;
- gerar relatórios administrativos;
- auxiliar atendimentos através de um chatbot virtual.

O sistema centraliza todas as informações em uma única plataforma digital, permitindo maior organização, rapidez e segurança no ambiente hospitalar.

## 🧠 Pensamento Computacional Aplicado

1. Decomposição

O sistema foi dividido em módulos menores:
  - autenticação;
  - consultas;
  - pacientes;
  - prontuários;
  - relatórios;
  - chatbot.

2. Reconhecimento de Padrões

Hospitais seguem padrões repetitivos:
  - cadastro;
  - atendimento;
  - armazenamento de informações;
  - geração de relatórios.

3. Abstração

Detalhes desnecessários foram removidos para focar apenas nos processos essenciais do sistema.

4. Algoritmos

O sistema utiliza lógica computacional para:
  - validar acessos;
  - verificar horários;
  - registrar informações;
  - organizar consultas.

## ⚠️ Desafios Encontrados

### Organização dos Módulos

O sistema possui diversas funcionalidades integradas, como:
  - pacientes;
  - médicos;
  - consultas;
  - exames;
  - relatórios.

- Solução:
Foi utilizada decomposição para dividir o sistema em módulos menores e organizados.


### Centralização das Informações

Hospitais geram muitos dados simultaneamente.

- Solução:
Foi proposto um banco de dados centralizado para armazenamento seguro e organizado.


### Controle de Agendamentos

Evitar conflitos de horários médicos.

- Solução:
Criação de lógica para verificação automática de horários disponíveis.


### Segurança de Dados

Informações médicas são altamente sensíveis.

- Solução:
Aplicação de autenticação e controle de acesso ao sistema.
